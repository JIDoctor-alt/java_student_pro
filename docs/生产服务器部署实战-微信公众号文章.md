## 三、核心步骤

### 1. 克隆代码

```bash
cd /opt/code-student-pro
git clone -b feature-develop \
  https://github.com/JIDoctor-alt/java_student_pro.git source
```

### 2. 构建后端 (Maven Wrapper)

服务器未安装 Maven，直接用项目内置的 `mvnw`：

```bash
cd /opt/code-student-pro/source/server/server
chmod +x mvnw
./mvnw clean package -DskipTests -Dmaven.test.skip=true -q
```

> 📦 构建产物: `target/server-0.0.1-SNAPSHOT.jar` (114MB)

### 3. 构建前端

```bash
cd /opt/code-student-pro/source/client
npm install
npm run build-only
```

> 📦 构建产物: `dist/` 目录（index.html + assets/）

### 4. 部署

```bash
# 停止服务
systemctl stop code-student-pro

# 备份旧文件
cp server.jar server.jar.bak
cp -r frontend frontend.bak

# 替换新文件
cp source/server/server/target/server-0.0.1-SNAPSHOT.jar server.jar
rm -rf frontend/*
cp -r source/client/dist/* frontend/

# 重启
systemctl restart code-student-pro
systemctl reload nginx
```

## 四、Systemd 服务配置

```ini
[Unit]
Description=CODE Student Pro Backend
After=network.target docker.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/code-student-pro
Environment=SPRING_PROFILES_ACTIVE=prod
Environment=DEEPSEEK_API_KEY=sk-xxxxxxxxx
Environment=MYSQL_PASSWORD=root
ExecStart=/usr/bin/java -jar /opt/code-student-pro/server.jar
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

## 八、常见问题与解决

### 8.1 测试编译失败

```
ERROR: package org.junit.jupiter.api does not exist
```

**解决**: 添加 `-Dmaven.test.skip=true` 跳过测试编译。

### 8.2 DEEPSEEK_API_KEY 未配置

```
ERROR: AI API Key 未配置：请设置 DEEPSEEK_API_KEY
```

**解决**: 在 systemd 服务文件中设置环境变量：

```bash
sed -i 's|Environment=DEEPSEEK_API_KEY=.*|Environment=DEEPSEEK_API_KEY=你的Key|' \
  /etc/systemd/system/code-student-pro.service
systemctl daemon-reload && systemctl restart code-student-pro
```

### 8.3 网络不稳定导致推送失败

GitHub HTTPS 推送间歇性超时，解决方案：

```bash
# 方案 1: 通过代理推送
git config http.proxy http://127.0.0.1:7890
git push

# 方案 2: 使用 SSH
git remote set-url origin git@github.com:user/repo.git
git push
```

## 十、总结

整个部署流程的核心要点：

1. **Maven Wrapper 救场** — 无需全局安装 Maven，直接用项目内的 `mvnw`
2. **Systemd 管理生命周期** — 自动重启、环境变量、日志一应俱全
3. **Docker 管理中间件** — MySQL + Redis 容器化，数据持久化到宿主机
4. **Nginx 前端静态文件 + API 代理** — 简洁高效
5. **备份先行** — 每次部署前备份旧的 JAR 和前端文件，随时可回滚

从克隆代码到服务上线，全程约 **3-5 分钟**（含构建时间）。

> 完整部署脚本已在项目 `deploy/` 目录中，支持一键自动化部署。
