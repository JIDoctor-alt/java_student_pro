# java_student_pro — 项目架构学习笔记

> AI 驱动的 NoCode 应用生成平台。用户描述需求 → AI 实时生成可运行的网页/Vue 工程 → 对话式迭代修改 → 一键部署。

---

## 1. 项目规模

| 指标 | 数值 |
|------|------|
| 总文件数 | ~223 |
| 后端 Java 文件 | ~90 |
| 前端 Vue/TS 文件 | ~45 |
| 配置文件 | ~15 |
| 数据库 SQL | 3 个表 |
| AI Prompt | 3 套 |

---

## 2. 技术栈

| 层级 | 技术 |
|------|------|
| **前端** | Vue 3 + TypeScript + Vite 6 + Pinia + Vue Router 5 + Ant Design Vue 4 |
| **后端** | Spring Boot 3.5.16 + Java 21 + Maven |
| **AI** | LangChain4j 1.17 (DeepSeek) + LangGraph4j 1.8.19 (工作流编排) |
| **ORM** | MyBatis Flex 1.11.7 |
| **数据库** | MySQL (`student_pro`) |
| **缓存/Session** | Redis (Spring Session + LangGraph Checkpoint) |
| **浏览器自动化** | Selenium 4.27 + WebDriverManager (封面截图) |
| **对象存储** | 腾讯云 COS (可选) |
| **API 文档** | Knife4j (OpenAPI 3) |
| **部署** | Python 自动化 + systemd + Nginx + Docker Compose |

## 5. 三大代码生成模式

### 5.1 HTML 模式 (`html`)

- **输入**: 用户自然语言描述
- **输出**: 单个 `index.html` 文件 (CSS 在 `<style>` 内, JS 在 `<script>` 内)
- **AI**: LangChain4j `@AiService` + `@SystemMessage` 结构化输出 `HtmlCodeResult`
- **流程**: Prompt 素材增强 → AI 生成 → 质量检测 → 落盘

### 5.2 多文件模式 (`multi_file`)

- **输入**: 用户自然语言描述
- **输出**: `index.html` + `style.css` + `script.js` 三个文件
- **AI**: LangChain4j `@AiService` + `@SystemMessage` 结构化输出 `MultiFileCodeResult`
- **流程**: 同 HTML 模式

### 5.3 Vue3 工程模式 (`vue_project`)

- **输入**: 用户自然语言描述
- **输出**: 完整 Vue3 + Vite 工程 → `npm install && npm run build` → `dist/`
- **AI**: LangChain4j Agent + Tool Calling (`saveFile` / `readFile` / `listFiles`)
- **流程**:
  1. Agent 思考 → 逐个 `saveFile` 写入工程文件
  2. 后端检测文件完整 → `npm install`
  3. `npm run build` → `dist/`
  4. 质量检测 → 封面截图

## 6. LangGraph4j 工作流详解

### 6.1 架构概览

代码生成工作流分为两个独立的 DAG (有向无环图)：

```
┌─────────────────────────────────────────────────────────┐
│  prepareGraph (准备阶段)                                 │
│                                                         │
│  START ──→ collect_assets ──→ enrich_prompt ──→ END     │
│             (素材收集)        (Prompt 增强)              │
├─────────────────────────────────────────────────────────┤
│  finalizeGraph (收尾阶段, 支持 Redis Checkpoint)         │
│                                                         │
│  START ──→ quality_check ──┬──[通过]──→ save_code ──→ END│
│             (质量检测)      │                           │
│                            └──[失败]──→ retry_generate  │
│                                        (重试生成)       │
│                                        └──→ quality_check (循环)│
└─────────────────────────────────────────────────────────┘
```

### 6.2 共享状态 `CodeGenWorkflowState`

继承 `AgentState`，包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| userMessage | String | 用户原始输入 |
| codeGenType | String | 生成类型 |
| appId | Long | 应用 ID |
| imageAssets | ImageAssetBundle | 收集的图片素材 |
| enrichedPrompt | String | 增强后的 Prompt |
| generatedCode | String | AI 生成的代码 |
| qualityReport | CodeQualityReport | 质量检测报告 |
| qualityPassed | Boolean | 是否通过 |
| retryCount | Integer | 当前重试次数 |
| workflowLog | List\<String> | 工作流日志 |

### 6.3 节点详解

| 节点 | 类 | 职责 |
|------|------|------|
| 素材收集 | `CollectAssetsNode` | 并发调用 Pexels + unDraw 搜索图片 |
| Prompt 增强 | `EnrichPromptNode` | 将素材 URL 列表注入 Prompt 末尾 |
| 质量检测 | `QualityCheckNode` | 调用 `CodeQualityChecker` 检测代码结构 |
| 重试生成 | `RetryGenerateNode` | 携带质量检测反馈重新调用 AI |
| 代码落盘 | `SaveCodeNode` | 解析代码并保存到文件系统 |

### 6.4 条件路由 `QualityCheckRouterEdge`

```java
if (qualityPassed) → "save_code"
if (Vue_PROJECT)  → "save_code"  // Vue 不走自动重试
if (retryCount < maxRetries) → "retry_generate"
else → "save_code"  // 超重试次数，强制落盘
```

### 6.5 Checkpoint 持久化

- `finalizeGraph` 支持 Redis Checkpoint (`langgraph4j-redis-saver`)
- 使用独立 Redis database (db2)，避免与 Session (db0) 冲突
- `RunnableConfig` 以 `threadId = "codegen-app-{appId}-{phase}"` 区分会话
- 支持断点恢复：工作流中断后可从上次状态继续

## 7. 图片素材收集系统

### 7.1 流程

```
用户输入 → ImageAssetCollector.collect()
  ├── 提取关键词 (英文优先, 中文映射: 电商→shopping, 教育→education, ...)
  ├── 并发调用:
  │   ├── PexelsClient.search()  → List<PhotoAsset>
  │   └── UndrawClient.search()  → List<IllustrationAsset>
  └── 合并 → ImageAssetBundle → toPromptBlock() 格式化为 Prompt 段落
```

### 7.2 Pexels 客户端

- API: `https://api.pexels.com/v1/search`
- 认证: `Authorization: {API_KEY}`
- 参数: query, per_page (1-10), locale=zh-CN
- 超时: 8 秒

### 7.3 unDraw 客户端

- 通过 Next.js data API 搜索: `https://undraw.co/_next/data/{buildId}/search/{keyword}.json`
- buildId 从首页 HTML 正则提取，缓存 1 小时
- 返回插画 slug → 拼出页面 URL 和 SVG URL

### 7.4 中文关键词映射

| 中文关键词 | 英文映射 |
|-----------|---------|
| 电商/商城/购物 | shopping |
| 教育/学习/课程 | education |
| 旅游/旅行 | travel |
| 美食/餐厅 | food |
| 科技/数据 | technology |
| 其他 | website |

## 8. 代码质量检测系统

### 8.1 检测维度 (HTML/多文件模式)

| 检测项 | 级别 | 规则 |
|--------|------|------|
| 内容非空 | issue | `generatedCode` 为空 |
| HTML 结构 | issue | 缺少 `<body>` 标签 |

## 10. SSE 流式通信

### 10.1 前端 SSE 消费 (`sse.ts`)

使用 `fetch` + `ReadableStream` (而非 `EventSource`)，优势:
- 可处理 401 JSON 错误响应 (EventSource 只能接收 text/event-stream)
- 无自动重连干扰
- 支持 `AbortController` 手动取消

```typescript
consumeSseStream({
  url,
  signal,
  onChunk: (chunk) => { /* 逐 token 处理 */ },
  onEvent: (eventName, data) => { /* 命名事件处理 */ },
  onDone: () => { /* 流结束 */ },
  onError: (msg) => { /* 错误处理 */ },
})
```

### 10.2 SSE 事件类型

| 事件名 | 数据 | 说明 |
|--------|------|------|
| message (默认) | AI 生成的代码片段 | 逐 token 输出 |
| tool-start | `saveFile → src/views/Home.vue` | Vue Agent 工具调用 |
| build-log | `vite v6.2.0 building for production...` | npm 构建日志 |
| quality-report | `质量检测通过` | 质量检测结果 |
| preview-ready | `ok` | Vue 构建完成，预览就绪 |
| cover-ready | `http://xxx/cover.jpg` | 封面截图完成 |
| gen-error | `生成失败：xxx` | 错误信息 |
| done | `[DONE]` | 流结束标记 |

### 10.3 两种 SSE 实现对比

| 模式 | 后端实现 | 前端消费 |
|------|----------|----------|
| HTML/多文件 | `Flux<ServerSentEvent<String>>` (Reactor) | `consumeSseStream()` |
| Vue 工程 | `SseEmitter` (Servlet) | `consumeSseStream()` |

## 11. Vue 工程 Agent 系统

### 11.1 工具调用 (Tool Calling)

`VueProjectFileTool` 提供 3 个 LangChain4j `@Tool`:

| 工具 | 参数 | 说明 |
|------|------|------|
| `saveFile` | path, content | 保存文件到项目目录 |
| `readFile` | path | 读取已有文件 (增量修改) |
| `listFiles` | dir | 列出目录文件 |

### 11.2 上下文绑定

使用 `InheritableThreadLocal<Long> APP_ID` 绑定当前生成任务的应用 ID，确保多用户并发时文件隔离。

### 11.3 安全防护

`resolveSafePath()` 防止路径遍历攻击:
```java
Path resolved = base.resolve(relativePath).normalize().toAbsolutePath();
if (!resolved.startsWith(base)) {
    throw new IllegalArgumentException("非法路径：" + relativePath);
}
```

### 11.4 构建流程

```
Agent saveFile 写入所有文件
  → VueProjectBuildService.build()
    → PackageNormalizer.normalize()  // 标准化 package.json
    → PreviewPathFixer.patchViteConfigBeforeBuild()  // 确保 base: './'
    → npm install
    → npm run build
    → PreviewPathFixer.patchDistIndexAfterBuild()  // 修正 dist 资源路径
    → 检测 dist/index.html 存在
```

### 11.5 过滤机制

`VueProjectCodegenExecutor.shouldSkipPartial()` 过滤工具调用 JSON，避免前端对话区展示乱码:
- 包含 `"path"` + `"content"` → 跳过
- 以 `saveFile` 开头 + `{` → 跳过
- 以 `{` 开头 + `"arguments"` → 跳过

## 12. 封面生成系统

### 12.1 流程

```
SSE 流完成
  → AppCoverService.generateCoverAsync()  // 异步，不阻塞 SSE
    → AppCoverScreenshotService.capturePreviewScreenshot()
      → WebDriverManager.chromedriver().setup()
      → ChromeOptions: --headless=new, --disable-gpu, --no-sandbox
      → driver.get(previewUrl)
      → 等待 document.readyState === "complete"
      → 额外等待 coverWaitSeconds (默认 3s)
      → TakesScreenshot.getScreenshotAs(FILE)
      → 保存到 CODE_COVER_ROOT_DIR/cover_{appId}.png
    → resolveCoverPublicUrl()
      → 优先: COS 上传 → 返回 COS URL
      → 回退: 本机静态目录 URL
    → 回写 app.cover 字段
```

### 12.2 SSE 推送封面 URL

Vue 工程模式下，封面 URL 通过 SSE `cover-ready` 事件推送:
```java
appCoverService.generateCoverAsync(appId, coverUrl -> {
    sendSseEvent(emitter, "cover-ready", coverUrl);
    sendSseEvent(emitter, "done", "[DONE]");
    emitter.complete();
});
```

### 13.3 前端路由守卫

`router.beforeEach()`:
1. 首次进入或刷新后，远程拉取用户信息 `loginUserStore.fetchLoginUser()`
2. 需要登录但未登录 → 跳转 `/user/login?redirect=...`
3. 需要管理员但不是管理员 → 提示无权限，跳转首页

## 18. AI System Prompt 设计

### 18.1 HTML Prompt 要点

- 纯原生 HTML/CSS/JS，禁止第三方框架
- 单文件输出
- `data-code-id` 属性要求 (可视化编辑)
- 图片素材例外: 仅可使用 Prompt 附带的 Pexels/unDraw URL
- 直接输出代码，禁止询问或解释

### 18.2 多文件 Prompt 要点

- 三文件分离: index.html / style.css / script.js
- HTML 正确引用 CSS/JS (相对路径)
- `data-code-id` 属性要求
- 其他同 HTML Prompt

### 18.3 Vue 工程 Prompt 要点

- Vue 3 + TypeScript + Vite 5 + Vue Router 4 + Pinia
- 固定依赖版本 (避免 AI 编造)
- `vite.config.ts` 必须 `base: './'`
- 工作方式: 思考 → 逐个 `saveFile` → 收尾说明
- 可视化编辑: 先 `readFile` 再精准修改
- 禁止事项: Vue 2 Options API、JSX/TSX、不存在的依赖
